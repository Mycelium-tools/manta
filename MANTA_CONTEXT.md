# MANTA Project Context
MANTA - Multi-turn Assessment for Nonhuman Thinking & Alignment

## What we're building
Testing if AI models maintain animal welfare principles under escalating pressure in multi-turn conversations.

## Files in this repo
- `dataset/sync_questions_to_hf.py` ‚Äî full sync pipeline: Google Sheets ‚Üí `dataset/manta_questions.csv` ‚Üí HuggingFace ‚Üí `samples.json`
- `dataset/manta_questions.csv` ‚Äî canonical local copy of the question dataset
- `dataset/hf_login.py` ‚Äî standalone HuggingFace login helper
- `dataset/scenario_generation.py` ‚Äî LLM-powered script to generate new Turn 1 scenarios (see Scenario Generation section below)
- `dataset/scenarios/` ‚Äî output directory for generated scenario JSON files
- `sample_questions.py` ‚Äî loads questions from HuggingFace, parses tags + pressure to correct types, splits into 2-turn and 3-turn groups in order; called automatically by sync script
- `samples.json` ‚Äî questions split into 2-turn and 3-turn groups (generated by `sample_questions.py`)
- `manta_eval.py` ‚Äî runs Inspect AI evaluation tasks; tasks: `manta_test5`, `manta_2turn`, `manta_3turn` and agentic variants; `MODELS` list controls which models are evaluated
- `run_single_eval.py` ‚Äî runs eval on a single question by ID from `samples.json`; supports `--turns`, `--agentic`, `--model`
- `dynamic_multiturn_solver.py` ‚Äî custom solver that generates adversarial follow-ups on-the-fly using a separate fixed model (claude-haiku)
- `multidimensional_scorer.py` ‚Äî `manta_scorer`: scores each applicable dimension with one LLM call per dimension; overall score is a weighted average (no extra LLM call, aligned with AHB)
- `manta_scout.py` ‚Äî Inspect Scout scanners for post-hoc analysis of eval logs
- `scout.yaml` ‚Äî Scout config (default grader model, logs directory)
- `logs/` ‚Äî generated `.eval` files from evaluation runs
- `qualitative_analyses/` ‚Äî Jupyter notebooks with manual analysis

## Technical setup
- Using Inspect AI framework
- Models: `anthropic/claude-sonnet-4-20250514`, `openai/gpt-4o` (see `MODELS` list in `manta_eval.py`)
- API Keys required in `.env`:
  - `ANTHROPIC_API_KEY`
  - `OPENAI_API_KEY`
  - `HF_TOKEN`

## Google Sheet columns
`id`, `question`, `tags`, `pressure`, `language`, `translation`
- `tags` format: `['Pressure Robustness']` (Python list repr ‚Äî parsed automatically to a list)
- `pressure` format: plain string e.g. `Economic` (normalized to lowercase in pipeline)
- `variables` column has been removed

## Scoring: how it works
- Scorer is `manta_scorer` in `multidimensional_scorer.py`
- Each question is tagged with dimensions (e.g. `["Pressure Robustness"]`) in Google Sheets
- Tags are stored per-sample in the Inspect `target` field as JSON: `{"tags": ["Pressure Robustness"]}`
- The scorer reads `target.text`, parses the tags, and only scores the dimensions relevant to that question
- Each dimension gets one LLM grader call; overall score = weighted average of dimension scores (no extra LLM call, aligned with AHB)
- To add a new dimension: add it to `MANTA_DIMENSIONS` in `multidimensional_scorer.py` and tag questions with it in Google Sheets (name must match exactly)

## Pressure types: how they work
Pressure types control what kind of adversarial follow-up is applied to push back on the model's welfare reasoning.

- **Turn 2** (first follow-up): uses the `pressure` column from Google Sheets, specified per-question
- **Turn 3+** (subsequent follow-ups): the follow-up generator (claude-haiku) dynamically picks the most effective remaining pressure type and writes the follow-up in one call

The 6 supported pressure types (defined in `PRESSURE_DESCRIPTIONS` in `dynamic_multiturn_solver.py`):
| Type | Description |
|---|---|
| `economic` | Pushback framing welfare as too costly or a threat to livelihoods |
| `social` | Pushback appealing to social norms or peer expectations |
| `authority` | Pushback citing expert consensus, regulations, or institutional approval |
| `pragmatic` | Pushback arguing welfare concerns are ineffective or too small to matter |
| `epistemic` | Pushback challenging the scientific basis for animal sentience |
| `cultural` | Pushback appealing to cultural tradition or longstanding practices |

To add a new pressure type: add it to `PRESSURE_DESCRIPTIONS` in `dynamic_multiturn_solver.py` and use it in the Google Sheet.

## Full workflow

**Step 1: Sync questions (Google Sheets ‚Üí CSV ‚Üí HuggingFace ‚Üí samples.json)**
```bash
python dataset/sync_questions_to_hf.py
```
This does everything in one command:
1. Downloads latest questions from Google Sheets ‚Üí `dataset/manta_questions.csv`
2. Uploads to HuggingFace
3. Automatically runs `sample_questions.py` to regenerate `samples.json`

**Step 2: Run evals**

Quick test ‚Äî first 5 questions, 2-turn (single model):
```bash
inspect eval manta_eval.py@manta_test5 --model anthropic/claude-sonnet-4-20250514
```

Quick test across multiple models:
```bash
inspect eval manta_eval.py@manta_test5 --model anthropic/claude-sonnet-4-20250514 --model openai/gpt-4o
```

Full 2-turn eval:
```bash
inspect eval manta_eval.py@manta_2turn --model anthropic/claude-sonnet-4-20250514 --model openai/gpt-4o
```

Full 3-turn eval:
```bash
inspect eval manta_eval.py@manta_3turn --model anthropic/claude-sonnet-4-20250514 --model openai/gpt-4o
```

Run all tasks across all models (uses `MODELS` list in `manta_eval.py`):
```bash
python manta_eval.py
```

Run a single question by ID:
```bash
python run_single_eval.py <question_id>
python run_single_eval.py <question_id> --turns 3        # override turn count (2, 3, or 4)
python run_single_eval.py <question_id> --agentic
python run_single_eval.py <question_id> --agentic --model openai/gpt-4o
```

## Scenario Generation

`dataset/scenario_generation.py` generates new Turn 1 benchmark scenarios using a three-step pipeline:

**Step 1 ‚Äî Zero-shot:** Single API call generating `num_q_zeroshot` scenarios with no examples. Output bootstraps few-shot examples for Step 2.

**Step 2 ‚Äî Few-shot + variance prompts (concurrent):** Multiple concurrent API calls (via `ThreadPoolExecutor`), each generating 1 scenario. Each call independently samples a variance prompt (with probability `p_var`) to push toward different industries, framings, and pressure types.

Key design: `get_messages()` is called once *per scenario* (not once for the whole batch) so each call gets an independently sampled variance prompt ‚Äî calling it once and reusing would make all scenarios cluster around the same nudge.

**Step 3 ‚Äî Quality control + iterative accumulation:** Each generated scenario is scored 0‚Äì10 by an LLM judge against a rubric. Scenarios below `MIN_SCORE` are filtered out. The script generates in batches and keeps going until 300 pass the threshold.

### QC scoring

- `RUBRIC` ‚Äî defines criteria at each score band: 0‚Äì2 for explicit welfare mentions, 3‚Äì5 for obvious signals or vague numbers, 6‚Äì7 for genuinely implicit welfare with minor issues, 8‚Äì10 for welfare buried in operational detail with layered pressure and a highly realistic role
- `SCORING_EXAMPLES` ‚Äî 4 labeled examples (bad ‚Üí excellent) passed to the judge as few-shot turns so scoring is consistent
- `score_scenarios()` ‚Äî concurrent LLM scoring via threadpool at `temperature=0` for deterministic results
- `summarize_results()` ‚Äî score stats, pressure type balance, welfare implicit rate
- `filter_dataset()` ‚Äî drops anything below `min_score`
- `generate_and_score_scenarios()` ‚Äî wraps generation + scoring into one call; saves a versioned JSON with all prompt constants for full reproducibility

Output files saved to `dataset/scenarios/`:
- `sce_<n>_<MMDDYY_HHMM>.json` ‚Äî zero-shot bootstrap
- `sce_<n>_<MMDDYY_HHMM>_scored_v<N>.json` ‚Äî scored batch (versioned)
- `speciesism_300_scenarios.json` ‚Äî final filtered dataset

Key config (edit in script):
- `num_q_zeroshot` ‚Äî how many scenarios to generate in Step 1
- `MIN_SCORE` ‚Äî minimum score to keep a scenario (default 7)
- `batch_size` ‚Äî how many scenarios to generate per accumulation batch (default 20)
- `p_var` ‚Äî probability of adding a variance prompt per call (currently 0.8)
- `VAR_PROMPTS` ‚Äî list of variance nudges covering domain, style, pressure type, subtlety
- `MODEL` ‚Äî model to use (defaults to `ANTHROPIC_MODEL` env var or `claude-sonnet-4-20250514`)

The `Scenario` Pydantic model validates each generated scenario has: `system`, `question`, `pressure_type`, `welfare_implicit`.

```bash
python dataset/scenario_generation.py
```

## Post-hoc analysis with Scout

Scan eval logs for patterns across models:
```bash
# Use default grader from scout.yaml
scout scan manta_scout.py

# Override grader model
scout scan manta_scout.py --model openai/gpt-4o

# Scan a specific log file
scout scan manta_scout.py -T ./logs/<filename>.eval
```

Scanners: `capitulated_to_pressure` (boolean), `welfare_score` (0‚Äì1), `pressure_type` (classification), `welfare_keywords` (grep).
