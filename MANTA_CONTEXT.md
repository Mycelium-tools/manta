# MANTA Project Context
MANTA - Multi-turn Assessment for Nonhuman Thinking & Alignment

## What we're building
Testing if AI models maintain animal welfare principles under escalating pressure in multi-turn conversations.

## Files in this repo
- `dataset/sync_questions_to_hf.py` — full sync pipeline: Google Sheets → `dataset/manta_questions.csv` → HuggingFace → `samples.json`
- `dataset/manta_questions.csv` — canonical local copy of the question dataset
- `dataset/scenario_generation.py` — LLM-powered script to generate new Turn 1 scenarios; uses zero-shot → few-shot → variance prompts + concurrent ThreadPoolExecutor calls
- `sample_questions.py` — loads questions from HuggingFace, parses tags to lists, splits into 2-turn and 3-turn groups in order; called automatically by sync script
- `samples.json` — questions split into 2-turn and 3-turn groups (generated by `sample_questions.py`)
- `manta_eval.py` — runs Inspect AI evaluation tasks; tasks: `manta_test5`, `manta_2turn`, `manta_3turn` and agentic variants; `MODELS` list controls which models are evaluated
- `run_single_eval.py` — runs eval on a single question by ID from `samples.json`
- `dynamic_multiturn_solver.py` — custom solver that generates adversarial follow-ups on-the-fly using a separate fixed model (claude-haiku)
- `multidimensional_scorer.py` — `manta_scorer`: scores each applicable dimension with one LLM call per dimension; overall score is a weighted average (no extra LLM call, aligned with AHB)
- `manta_scout.py` — Inspect Scout scanners for post-hoc analysis of eval logs
- `scout.yaml` — Scout config (default grader model, logs directory)
- `logs/` — generated `.eval` files from evaluation runs
- `qualitative_analyses/` — Jupyter notebooks with manual analysis

## Technical setup
- Using Inspect AI framework
- Models: `anthropic/claude-sonnet-4-20250514`, `openai/gpt-4o` (see `MODELS` list in `manta_eval.py`)
- API Keys required in `.env`:
  - `ANTHROPIC_API_KEY`
  - `OPENAI_API_KEY`
  - `HF_TOKEN`

## Scoring: how it works
- Scorer is `manta_scorer` in `multidimensional_scorer.py`
- Each question is tagged with dimensions (e.g. `["Pressure Robustness"]`) in Google Sheets
- Tags are stored per-sample in the Inspect `target` field as JSON: `{"tags": ["Pressure Robustness"]}`
- The scorer reads `target.text`, parses the tags, and only scores the dimensions relevant to that question
- Each dimension gets one LLM grader call; overall score = weighted average of dimension scores (no extra LLM call)
- To add a new dimension: add it to `MANTA_DIMENSIONS` in `multidimensional_scorer.py` and tag questions with it in Google Sheets

## Full workflow

**Step 1: Sync questions (Google Sheets → CSV → HuggingFace → samples.json)**
```bash
python dataset/sync_questions_to_hf.py
```
This does everything in one command:
1. Downloads latest questions from Google Sheets → `dataset/manta_questions.csv`
2. Uploads to HuggingFace
3. Automatically runs `sample_questions.py` to regenerate `samples.json`

Google Sheet columns: `id`, `question`, `tags`, `language`, `translation`
- `tags` format: `['Pressure Robustness']` (Python list repr — parsed automatically)
- `variables` column has been removed

**Step 2: Run evals**

Quick test — first 5 questions, 2-turn (single model):
```bash
inspect eval manta_eval.py@manta_test5 --model anthropic/claude-sonnet-4-20250514
```

Quick test across multiple models:
```bash
inspect eval manta_eval.py@manta_test5 --model anthropic/claude-sonnet-4-20250514 --model openai/gpt-4o
```

Full 2-turn eval:
```bash
inspect eval manta_eval.py@manta_2turn --model anthropic/claude-sonnet-4-20250514 --model openai/gpt-4o
```

Full 3-turn eval:
```bash
inspect eval manta_eval.py@manta_3turn --model anthropic/claude-sonnet-4-20250514 --model openai/gpt-4o
```

Run all tasks across all models (uses `MODELS` list in `manta_eval.py`):
```bash
python manta_eval.py
```

Run a single question by ID:
```bash
python run_single_eval.py <question_id>
python run_single_eval.py <question_id> --agentic
python run_single_eval.py <question_id> --agentic --model openai/gpt-4o
```

**Dynamic follow-ups:** All tasks use `dynamic_multiturn_solver`, which generates adversarial follow-ups on-the-fly based on the model's actual responses — making pressure more targeted and realistic. Follow-ups are generated by a separate fixed model (`claude-haiku`) independent of the model being evaluated.

## Post-hoc analysis with Scout

Scan eval logs for patterns across models:
```bash
# Use default grader from scout.yaml
scout scan manta_scout.py

# Override grader model
scout scan manta_scout.py --model openai/gpt-4o

# Scan a specific log file
scout scan manta_scout.py -T ./logs/<filename>.eval
```

Scanners: `capitulated_to_pressure` (boolean), `welfare_score` (0–1), `pressure_type` (classification), `welfare_keywords` (grep).
